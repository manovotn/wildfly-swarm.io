    • Announcing WildFly Swarm 2017.8.1

    By The WildFly Swarm Team | Published Aug 2, 2017

    WildFly Swarm 2017.8.1

    On this day in 1914, the first steamer passed through the Panama Canal. Cool, eh?

    On this day in 2017, we’re pleased to announce the 2017.8.1 release of WildFly Swarm.

    What’s New?

    • AsciiDoctorJ

    • Jaeger & OpenTracing

    • Bug fixes

    Changelog

    Release notes for 2017.8.1 are available here.

    Bug

    • [SWARM-1467] "runtime" modules should be marked private (Done)

    • [SWARM-1466] fail fast when unable to create default deployment (Done)

    • [SWARM-1462] Unable to access Swagger webapp UI (Done)

    • [SWARM-1460] HttpSecurityPreparer should not ignore unknown auth methods (Done)

    • [SWARM-1458] javax.xml.transform.TransformerConfigurationException: Translet class loaded, but unable to create translet instance. (Done)

    • [SWARM-1449] Fix ReflectionUtil.getMethodsWithAnnotation() (Done)

    • [SWARM-1447] default datasource not set in the EE subsystem (Done)

    • [SWARM-1224] Cannot use XSLT: TransformerFactory.newTemplates(…​) fails with "Translet class loaded, but unable to create translet instance." (Done)

    Task

    • [SWARM-1456] Remove bom-dependency as it’s not needed (Done)

    • [SWARM-1441] Update all Boosters to 2017.7.0 (Done)

    • [SWARM-1440] Separate build profiles (Done)

    • [SWARM-1434] Move depMan dependencies into unsupported profile as needed (Done)

    • [SWARM-1372] Gather License data for dependencies (Done)

    Feature Request

    • [SWARM-1435] Introduce an AsciidoctorJ fraction (Done)

    • [SWARM-1411] Add Jaeger fraction (Done)

    • [SWARM-1404] Add Servlet framework integration to OpenTracing fraction (Done)
