    • Announcing WildFly Swarm 2017.6.1

    By The WildFly Swarm Team | Published Jun 18, 2017

    WildFly Swarm 2017.6.1

    "Anyone who has never made a mistake has never tried anything new." (Albert Einstein)

    Albert is right. We tried something new. It broke the dependency management, so we changed it again. For the better. This is the short story of why we have 2017.6.1.

    What’s New?

    • SWARM-1369 caused trouble with project dependencies.

    SWARM-1369

    A change to the Wildfly Swarm BOM’s caused trouble in user projects, for instance the RESTeasy client errors documented in SWARM-1381. The BOM dependency management has been changed to fix this.

    Changelog

    Release notes for 2017.6.1 are available here.

    Task

    • [SWARM-1394] remove main() from our standalone-servers. (Done)

    • [SWARM-1386] Handle Maven mirrors better (Done)

    • [SWARM-1380] Investigate Examples CI Failures (Done)

    • [SWARM-1373] Print warning when mainClass property set (Done)

    Bug

    • [SWARM-1369] User projects picks up different dependent versions than WF Swarm build (Done)

    • [SWARM-1384] Keycloak standalone server seems to use deprecated main() (Done)

    • [SWARM-1381] RESTEasy Client doesn’t work with Swarm 2017.6.0 (Done)
